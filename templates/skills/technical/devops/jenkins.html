Jenkins


Jenkins is an open source continuous integration tool written in Java. The project was forked from Hudson after a dispute with Oracle.
Jenkins provides continuous integration services for software factory. It is a server-based system running in a servlet container such as Apache Tomcat or with a embed server.
It supports SCM tools such as Git, CVS, Subversion, ... and can execute Apache Ant and Apache Maven based projects as well as arbitrary shell scripts and Windows batch commands.
Build can be started by various means, including being triggered by commit in a version control system, scheduling via a cron-like mechanism, building when other builds have completed, by requesting a specific build URL ...

With Jenkins, I could set up multiple continuous integration workflow for quite varied technologies like java, php, python, nodejs, javascript ...
Also I write generally jenkins job with tasks fabric to have no code in the CI and you can easily migrate to another CI if necessary.
I Jenkins also expanded with the use of the docker , when a job was a container Started was launched to run tasks in a secluded and clean environment.
I could link Jenkins to scm systems such as Github.

Jenkins point of view

Jenkins makes it possible an integration server more or less rapid to implement depending on the choice of target architecture (standalone servlet server, slave master ...).
This implies a more or less administrative costs depending on the architecture and integrated plugins.
It has the advantage of being flexible and allowing suddenly able to really personalize their continuous integration system.
There is a strong community that creates and maintains plugins that will allow the incorporation of tools and adding functionality, such as slack plugins for sending the result of a job on a particular channel.
With jenkins, it is possible to create workflows according with the resulat of a job.
The risk with this tool is leaving the scope of continuous integration which amounts to :
  • Builder (build, compile ... software).
  • Test.
  • Packaged if all previous steps have worked.
  • Delivered a package into an artifact repository.

Compared to its concucents, Jenkins has the advantage of being usable in its infrastructure without firing a license but it can also be used under the Software as a service format.
Jenkins has a lot of competitor generally paid (either a license or format Software as Service), they have the advantage that there is no set up and administer the servuer continuous integration.
