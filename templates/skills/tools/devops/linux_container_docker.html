Docker


Docker est un logiciel open source qui permet d'embarquer une application dans un système de fichiers complet qui contient tout ce dont il a besoin pour s'exécuter:
  • Code
  • Outils système
  • Librairie
  • Environnement
Ceci garantit qu'il sera toujours exécuter de la même façon, quel que soit l'environnement, où il est exécuté.
Docker a développé tout un écosystème autour du "docker engine" comme :
  • Docker compose: permet de décrire un envrionnement complet avec un fichier yaml et de lever cet envrionnement et de faire une mise à l'échelle
  • Docker swarm: permet de gérer un cluster de docker engines afin de déployer du container sans se soucier d'où il va être déployé
  • Docker machine: permet de créer des vm avec des docker engines, principalement utilisé pour des environnements windows et mac
  • Docker cloud: une solution de déploiement de container orrientée cloud
  • Libnetwork: la solution de SDN (Software Defined Network) native à docker, ce qui permet de créer la couche réseau inter host.
Avec Docker j'ai pu faire :
  • J'ai packagé des applications comme des api rest, des applications sur "message queue", GPU et des outils tel que firefox
  • J'ai utilisé Docker dans des intégrations continues afin de pouvoir exécuter les tests et les builds dans des environnements isolés et "propres"
  • J'ai déployé une architecture microservices (40 microservices) sur un cluster de docker engines avec swarm et docker cloud.
  • J'ai pu faire des POC sur les solutions de SDN autour de docker (weave et libnetwork)
  • J'ai créé des environnements de développement quasiment iso à la production
